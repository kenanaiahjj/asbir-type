# “What’s New” 1.1.0 Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Add an automatically opening, dismissible 1.1.0 release panel to the Asbir Type Studio website.

**Architecture:** Keep the release announcement inside the existing single-page `render()` output. A small versioned UI state object controls open/closed visibility, `localStorage` remembers dismissal, and the existing `bind()` lifecycle wires the trigger, close button, Escape key, and download links. CSS adds a fixed bottom-right dock that becomes a bottom inset sheet on narrow screens.

**Tech Stack:** Vanilla JavaScript, existing CSS tokens, Python `unittest` source contracts, Vite.

## Global Constraints

- The panel opens automatically only when `asbir-whats-new-1.1.0-dismissed` is absent from browser storage.
- Dismissal never blocks page scrolling and never uses `aria-modal="true"`.
- Copy must describe both Sans and Mono 1.1.0 additions and link to `/downloads/AsbirSans-1.1.0.zip` and `/downloads/AsbirMono-1.1.0.zip`.
- The trigger remains available after dismissal and the panel must close with Escape.
- Motion must have a `prefers-reduced-motion: reduce` fallback.
- Do not modify the unrelated Serif-generated files or the pre-existing `AGENTS.md` edit.

import unittest
from pathlib import Path


ROOT = Path(__file__).resolve().parents[1]


class WhatsNewPanelTests(unittest.TestCase):
    def setUp(self):
        self.source = (ROOT / 'src' / 'main.js').read_text()

    def test_panel_has_versioned_copy_links_and_storage_key(self):
        for expected in (
            "What’s new · 1.1.0",
            'asbir-whats-new-1.1.0-dismissed',
            '/downloads/AsbirSans-1.1.0.zip',
            '/downloads/AsbirMono-1.1.0.zip',
            'True italic family',
            'WOFF2 + CSS kit',
            'Approved production family',
            'Fixed-cell terminal companion',
        ):
            self.assertIn(expected, self.source)

        self.assertIn('aria-expanded', self.source)
        self.assertIn("event.key === 'Escape'", self.source)
        self.assertIn('localStorage.setItem', self.source)


if __name__ == '__main__':
    unittest.main()

### Task 2: Add the panel state, markup, and bindings

**Files:**
- Modify: `src/main.js` near the existing `state` object, `render()`, and `bind()`.

**Interfaces:**
- Consumes: release download paths and existing render/bind lifecycle.
- Produces: `renderWhatsNew()` markup and persistent `whatsNewState` used by the CSS panel.

- [ ] **Step 1: Add the versioned state and storage helper**

Add immediately after the existing `state` declaration:

```js
const WHATS_NEW_VERSION = '1.1.0';
const WHATS_NEW_STORAGE_KEY = `asbir-whats-new-${WHATS_NEW_VERSION}-dismissed`;
const whatsNewState = {
  open: (() => {
    try { return localStorage.getItem(WHATS_NEW_STORAGE_KEY) !== '1'; }
    catch { return true; }
  })(),
};
```

- [ ] **Step 2: Add the release-panel renderer**

Add `renderWhatsNew()` before `render()`; it returns a fixed dock containing a trigger, an `aside` panel, the Sans and Mono rows, direct ZIP links, and a close button. The panel uses `hidden` when `whatsNewState.open` is false and sets the trigger’s `aria-expanded` to the same state.

- [ ] **Step 3: Mount the renderer in the existing app output**

Append `${renderWhatsNew()}` beside the existing toast element at the end of `app.innerHTML` so every existing `render()` call preserves the current release-panel state.

- [ ] **Step 4: Bind trigger, close, Escape, and dismissal**

In `bind()`, wire `#whats-new-trigger` and `#whats-new-close` to update `whatsNewState.open` and call `render()`. Extend the existing `document.onkeydown` handler so Escape closes the panel and writes `localStorage.setItem(WHATS_NEW_STORAGE_KEY, '1')`; keep the existing `G` guide shortcut intact.
